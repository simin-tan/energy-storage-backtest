# Battery Storage Arbitrage Backtest

A small Python pipeline that backtests a simple battery arbitrage strategy against
hourly electricity price data: charge when prices are low, discharge when prices
are high, and see what the simulated revenue looks like.

The price data used here is **synthetic** (generated by `setup_mock_data.py`), not
pulled from a live market. The point of this project was to get the data pipeline
correct — not to produce a real trading result.

## What it does

1. **`setup_mock_data.py`** — generates 7 days of mock hourly electricity price
   data and writes it to a local SQLite database (`energy_market.db`), including
   a deliberately injected duplicate timestamp and a missing value, to exercise
   the cleaning logic below.
2. **`pipeline.py`** — pulls the data via SQL, cleans it, runs the backtest, and
   produces an interactive HTML report:
   - Converts timestamps to `Europe/Berlin` local time, explicitly handling
     daylight saving time transitions (ambiguous hours during the October
     clock-back, non-existent hours during the March clock-forward), before
     converting to UTC for consistent downstream processing.
   - Averages out duplicate price readings at the same timestamp rather than
     arbitrarily keeping one.
   - Fills missing prices via linear interpolation rather than dropping the
     hour outright, to keep the time series continuous.
   - Runs a quantile-based charge/discharge strategy (bottom 25% price =
     charge, top 25% = discharge), applying an 85% round-trip efficiency
     factor to reflect real battery losses.
   - Exports an interactive Plotly chart (`battery_performance_report.html`)
     showing price over time with charge/discharge points marked.

## Setup

```bash
pip install -r requirements.txt
```

## Running it

```bash
# 1. Generate the mock database (only needed once, or to regenerate)
python setup_mock_data.py

# 2. Run the pipeline: clean data, backtest, generate report
python pipeline.py
```

This will print row counts and total simulated profit to the console, and write
`battery_performance_report.html` — open it in a browser to view the interactive
chart.

## Notes

- `energy_market.db` is generated locally and is excluded from version control
  via `.gitignore` — you'll need to run `setup_mock_data.py` first before
  `pipeline.py` will find any data.
- This is a portfolio/learning project, not a production trading system. Next
  step would be swapping the mock data source for a real market data API.
