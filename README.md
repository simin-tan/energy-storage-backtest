# Battery Storage Arbitrage Backtest

A small Python pipeline that backtests a simple battery arbitrage strategy against
hourly electricity price data: charge when prices are low, discharge when prices
are high, and estimate the simulated revenue.

The price data used here is **synthetic** (generated by `setup_mock_data.py`), not
pulled from a live market. 
**[View the live sample report →](https://simin-tan.github.io/energy-storage-backtest/battery_performance_report.html)**

The point of this project was to get the data pipeline
correct — not to produce a real trading result.

## What it does

1. **`setup_mock_data.py`** — generates 7 days of synthetic hourly electricity price
   data and writes it to a local SQLite database (`energy_market.db`), including
   a deliberately injected duplicate timestamp and a missing value, to exercise
   the data cleaning workflow.

2. **`src/data_loader.py`** — loads electricity price data from SQLite and
   performs data quality checks:
   - Drops rows with missing (null) prices.
   - Converts timestamps to `Europe/Berlin` local time, explicitly handling
     daylight saving time transitions (ambiguous hours during the October
     clock-back, non-existent hours during the March clock-forward), before
     converting to UTC for consistent downstream processing.
   - Averages duplicate price readings at the same timestamp rather than
     arbitrarily keeping one.
   - Drops any row left with an unresolved (NaT) timestamp after the DST
     handling above, logging how many rows were removed.

3. **`src/strategy.py`** — applies a simple battery arbitrage strategy:
   - Charges when prices fall into the lowest 25% of the observed range,
     discharges when prices rise into the highest 25% of the observed range.
   - Simulates a 1 MWh / 1 MW battery, respecting power limits and remaining
     capacity at each time step (energy moved per step = power × `interval_hours`).
   - Applies round-trip efficiency (85%) as separate charging and discharging
     losses (√0.85 each), rather than a single factor, so the battery can
     reach its full rated capacity.
   - `interval_hours` must match the actual time resolution of the input data
     (currently hourly); it is not inferred automatically.

4. **`src/report.py`** — generates an interactive Plotly report
   (`battery_performance_report.html`) showing electricity prices with
   charge/discharge events marked.

5. **`src/main.py`** — orchestrates the complete workflow from data loading to
   strategy execution and report generation.

## Setup

```bash
pip install -r requirements.txt
```

## Running it

```bash
# 1. Generate the mock database (only needed once, or to regenerate)
python setup_mock_data.py

# 2. Run the battery backtesting pipeline
python src/main.py
```

This will load and clean the electricity price data, run the battery arbitrage
strategy, print row counts and simulated profit to the console, and write
`battery_performance_report.html` — open it in a browser to view the interactive
chart.

## Notes

- `energy_market.db` is generated locally and is excluded from version control
  via `.gitignore` — you'll need to run `setup_mock_data.py` first before running
  `src/main.py`.
- This is a portfolio/learning project, not a production battery optimization system. 
- The next step would be replacing the synthetic electricity price data with real market data.
- Charge/discharge thresholds (25th/75th percentile) are computed over the
  entire dataset passed in, not recalculated per calendar week. This matches
  the current 7-day mock dataset but would need revisiting if the pipeline
  is ever fed multiple weeks of data.
