# Battery Storage Arbitrage Backtest

A small Python pipeline that backtests a simple battery arbitrage strategy against
hourly electricity price data: charge when prices are low, discharge when prices
are high, and estimate simulated revenue.

The price data used here is **synthetic** (generated by `scripts/setup_mock_data.py`), not
pulled from a live market. 
**[View the live sample report →](https://simin-tan.github.io/energy-storage-backtest/outputs/battery_performance_report.html)**

The focus of this project was building the data pipeline and workflow required
to support the backtesting process, rather than producing a real-world trading
result.

## Key Engineering Features

- **Timezone and daylight saving time handling** — Processes timestamps using the `Europe/Berlin` timezone, explicitly resolving DST-related ambiguous and non-existent hours before converting to UTC for consistent downstream analysis.

- **Data quality processing** — Handles timezone conversion and DST edge cases before normalizing timestamps to UTC for consistent downstream analysis.

- **Modular pipeline structure** — Separates data ingestion, cleaning, strategy execution, and reporting into independent Python modules, making the workflow easier to maintain and extend.

- **Reproducible backtesting workflow** — Provides a complete local pipeline that generates synthetic market data, runs the battery strategy, and produces an interactive report with a single command (`python src/main.py`).
  
## What it does

1. **`scripts/setup_mock_data.py`** — generates 7 days of synthetic hourly electricity price
   data and writes it to a local SQLite database (`data/energy_market.db`), including
   a deliberately injected duplicate timestamp and a missing value, to exercise
   the data cleaning workflow.

2. **`src/data_loader.py`** — loads electricity price data from SQLite and
   performs data quality checks:
   - Converts timestamps to `Europe/Berlin` local time, explicitly handling
     daylight saving time transitions (ambiguous hours during the October
     clock-back, non-existent hours during the March clock-forward), before
     converting to UTC for consistent downstream processing.
   - Averages duplicate price readings at the same timestamp rather than
     arbitrarily keeping one.
   - Fills missing prices via linear interpolation rather than dropping the
     hour outright, keeping the hourly time series continuous.

3. **`src/strategy.py`** — applies a simple battery arbitrage strategy:
   - Charges when prices fall into the lowest 25% of the observed range.
   - Discharges when prices rise into the highest 25% of the observed range.
   - Applies an 85% round-trip efficiency factor to account for battery losses.

4. **`src/report.py`** — generates an interactive Plotly report
   (`outputs/battery_performance_report.html`) showing electricity prices with
   charge/discharge events marked.

5. **`src/main.py`** — runs the complete workflow from data loading to
   strategy execution and report generation.

## Setup

```bash
pip install -r requirements.txt
```

## Running it

```bash
# 1. Generate the mock database (only needed once, or to regenerate)
python scripts/setup_mock_data.py

# 2. Run the battery backtesting pipeline
python src/main.py
```

This will load and clean the electricity price data, run the battery arbitrage
strategy, print row counts and simulated profit to the console, and write
`outputs/battery_performance_report.html` — open it in a browser to view the interactive
chart.

## Notes

- `data/energy_market.db` is generated locally and is excluded from version control
  via `.gitignore` — you'll need to run `scripts/setup_mock_data.py` first before running
  `src/main.py`.
- This is a portfolio/learning project, not a production battery optimization system.
- Future improvements would include integrating real market price data and extending the strategy with more advanced battery optimization constraints.
